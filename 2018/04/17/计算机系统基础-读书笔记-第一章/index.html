1 基础

QUESTION: HOW TO BUILD A NETWORK

计算机网络的主要特征是通用性。

得到网页的过程模型：

  • 域名解析(应用了UDP) => 建立TCP连接 => 浏览器发出请求，服务器返回页面 => 关闭TCP

构建计算机网络需要：

  • 确定影响网络设计的所有需求和约束。

    • 应用程序员：所需服务
    • 网络设计者： 高效
    • 网络提供者：易于执行管理

  • 连通性

    • 网络组件：节点、链路
    • 方式（从易到难）
      • 直连（一条链路，一条总线）
      • 间接连结（更多的链路，交换网络（常见类型：电路交换和分组交换），交换机（存储和转发分组））
      • 网络互联（多个网络，路由器/网关）
    • 网络互联关键
      • 为网络中每一个可到达的节点定一个地址，从而利用这个进行路由选择然后正确传送消息。

  • 有效性

    • 多路复用：多用户共享系统资源

      • 同步时分多路复用：STDM

      • 频分多路复用： FDM

      • 统计多路复用（按需而不是时间片的STDM）

        STDM：相当于传送带的固定位置占了固定的人，当接收数据的盒子经过一个人A之后，盒中数据被取出并到达下一个地方B，B装载数据，等待一个时间片的时间，再取出数据使下一个人C装载数据。其中ABC分别装载数据的时间是固定的一个时间片。给每个主机分配一段时间。

        FDM ：在传输数据的盒子中给每个主机分配一个位置，每个位置代表一个频率。

        以上两种有 时间片空闲或者位置空闲的情况，浪费资源。

        统计多路复用： 按照时间共享物理链路，传送带只有一个位置，需要的人就上前使用这个位置，并在规定的时间片内装载数据，接着数据传送给交换机，然后换下来给另外有需要的人用。

        其中，为了保证所有流都有机会传送数据，每个流在指定时间内只能传送一定大小的数据块（分组 ），超出则在这段时间都没有机会上前了。因此还需要主机（源节点）将消息化成几个分组，接收者重组还原消息。

        位置的使用权：以分组接分组为基础决定下一个发送分组的流。也相当于轮流发。

        这种方法可以避免空闲时间，使分组交换具有高效。

  • 使连接成功的主机上运行的应用程序有效通信

    • 请求/应答信道：保证数据保密完整–文件传输
    • 消息流信道：保证顺序不保证数据完整性，支持多点播送–视频点播
    • 可靠性
      • 从某些故障中恢复的能力

  • 1.3 网络体系结构

  • 协议：构成网络系统各层的抽象对象

    • 服务接口
    • 对等接口

  • OSI体系结构（7层）

    • 数据形式：原始比特（physical层）=>帧（dat a link -frame）=>分组(network -packet)=>消息(transport -message)

  • 因特网体系结构（TCP/IP体系结构)

    • TCP: Transmission Control Protocol
    • UDP: User Datagram Protocol
    • IP是结构的核心

  • 1.5 性能

    • 度量方法：带宽（吞吐量throughput）和时延（latency） 相对重要性依赖于应用
    • Latency = propagation +Trnsmit + Queue
    • Propagation = Distance / SpeedOfLight
    • Transmit = Size / BandWidth
    • Distance:数据需要穿越的线路长度
    • Size: 分组大小
    • SpeedOfLight : 光在线路中的有效速度

    小结

  • 需求分析 → 系统设计

    1. 连通性
    2. 可扩展连通性
    3. 通信 → 协议
    4. 易管理 → 分层体系结构

  • 已解决的问题

    • 五层的分层模型的具体传输过程是怎样的？
      • 发送数据时从上往下依次加头（需要尾部的加尾部（链路层））
      • 读取数据时从下至上，当前层剥去当前层相关的头部（如有尾部则剥去微博）取回数据部分，并上交给上一层。从Head2到Head5。
