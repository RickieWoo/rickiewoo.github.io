instanceof 和constructor 判断的变量，必须在当前页面声明的，比如，一个页面（父页面）有一个框架，框架中引用了一个页面（子页面），在子页面中声明了一个ary，并将其赋值给父页面的一个变量，这时判断该变量，Array == object.constructor;会返回false；

1、array属于引用型数据，在传递过程中，仅仅是引用地址的传递。

try-catch

Posted on 2018-05-23 Edited on 2020-06-08 In keep-learning
  • 使用 try catch 的使用无论是在 try 中的代码还是在 catch 中的代码性能消耗都是一样的。
  • 需要注意的性能消耗在于 try catch 中不要直接塞进去太多的代码（声明太多的变量），最好是吧所有要执行的代码放在另一个 function 中，通过调用这个 function 来执行。

针对第二点，可以查看 ECMA 中关于 try catch 的解释，在代码进入 try catch 的时候 js引擎会拷贝当前的词法环境，拷贝的其实就是当前 scope 下的所有的变量。

在使用 try catch 的时候尽量把 try catch 放在一个相对干净的 scope 中，同时在 try catch 语句中也尽量保证足够少的变量，最好通过函数调用方式来 try catch。

八分音符酱游戏设计与逻辑

Posted on 2018-05-13 Edited on 2020-06-08 In Magic

游戏设计与逻辑

首先开机后会进入一个开机画面，点击OK确认后，进入菜单选择，分别是键控和声控两个选项，根据选择显示键控（或声控）被选择的画面，确定选择之后点击OK即可正式开始游戏。

游戏的状态控制：初始化的时候记录菜单选择，游戏进行过程中根据玩家的选择控制中断中的行为，每一帧进行人物位置判断的时候，会结合地图位置来进行游戏状态判断，当人物达到死亡条件时，游戏不再进行，蜂鸣器报警并显示分数画面，玩家可选择重新开始。

游戏的计分系统：记录玩家存活时间作为游戏的分数，在本逻辑中每过一帧计一分。

游戏的初始地图是一个固定高度，宽度有限制随机的矩形数组集合，人物的位置也是随机（算法确保人物初始落在地面上，且距离“悬崖”较远），然后通过玩家在键控模式的按键中断，或者声控模式达到一定阈值的中断，结合人物的位置，来判断是否进行跳跃。其中，人物移动的每一帧都会结合地图及是否有中断信息来判断人物的位置、游戏的状态、同时使地图向前刷新。简化后的游戏状态模型可以参见游戏状态图。

Elastic Search 学习时看到的story

Posted on 2018-05-13 Edited on 2020-06-08 In Magic

许多年前，一个刚结婚的名叫 Shay Banon 的失业开发者，跟着他的妻子去了伦敦，他的妻子在那里学习厨师。 在寻找一个赚钱的工作的时候，为了给他的妻子做一个食谱搜索引擎，他开始使用 Lucene 的一个早期版本。

直接使用 Lucene 是很难的，因此 Shay 开始做一个抽象层，Java 开发者使用它可以很简单的给他们的程序添加搜索功能。 他发布了他的第一个开源项目 Compass。

后来 Shay 获得了一份工作，主要是高性能，分布式环境下的内存数据网格。这个对于高性能，实时，分布式搜索引擎的需求尤为突出， 他决定重写 Compass，把它变为一个独立的服务并取名 Elasticsearch。

第一个公开版本在2010年2月发布，从此以后，Elasticsearch 已经成为了 Github 上最活跃的项目之一，他拥有超过300名 contributors(目前736名 contributors )。 一家公司已经开始围绕 Elasticsearch 提供商业服务，并开发新的特性，但是，Elasticsearch 将永远开源并对所有人可用。

据说，Shay 的妻子还在等着她的食谱搜索引擎…
